I
แนวคิดหลัก

บทสรุปผู้บริหาร

ไอเดียกาแฟสองตัว — เครื่องสแกนคุณภาพ + ที่ปรึกษาสายพันธุ์ — รวมเป็นแอปเดียว เพราะให้บริการชาวสวนคนเดียวกัน คนละจังหวะของวงจรเดียวกัน.

  1. ตลาดกาแฟพิเศษไทยมี ส่วนต่างราคา ~10–16 เท่า จากเมล็ดเขียวถึงรีเทล — ชาวสวนได้ราว 10% ของปลายทาง.

  2. เกิดจากปัญหาแก้ได้ 3 อย่าง: ไม่มีตรวจคุณภาพระดับฟาร์ม · ไม่มีช่องขายตรงให้ roaster · ไม่มีตัวช่วยปลูกใหม่ให้ทนอากาศ.

  3. CV ตรวจตำหนิเริ่มเป็นของสามัญแล้ว — กำแพงจริงคือ ชุดข้อมูลไทย/ดอยช้าง + ความสัมพันธ์ชาวสวน + network effect ไม่ใช่ "เรามี AI".

  4. คุมทั้งวงจร ไม่ใช่จุดเดียว — ทำให้ป้องกันคู่แข่งที่ทำเฉพาะจุดเดียวได้ และสร้างวงจรสะสมข้อมูลที่ผลิตภัณฑ์เดี่ยวทำไม่ได้.

3 รากของปัญหา

  • ฝ่ายผู้ซื้อรู้คุณภาพมากกว่าผู้ขาย — ชาวสวนไม่รู้คะแนน SCA/ตำหนิของเมล็ดตัวเอง จึงเป็นผู้รับราคา · computer vision แก้.
  • ไม่มีช่องทางขายตรง — กฎหมายส่งออก จ่ายเงินข้ามชาติ กำแพงภาษา ขวางการขายตรงให้ roaster ต่างประเทศ · marketplace แก้.
  • ปลูกผิดสายพันธุ์ใน 5 ปี — อากาศเลื่อนเขตปลูกขึ้นเขา ~30–40 ม./ทศวรรษ (~+300–500 ม. ภายใน 2050, SSP2-4.5) แปลงต่ำกว่า ~1,300 ม. เสี่ยงเสื่อม · ที่ปรึกษาสายพันธุ์แก้.

ทำไมต้องเป็นปี 2026 — เน็ตไทยเข้าถึง ~95%, PromptPay ข้ามอาเซียนพร้อม, ระเบียบเร่งสัญชาติชาวเขามีผล 30 มิ.ย. 2025 ลด อุปสรรค KYC ของบัญชีธนาคารชาวสวน.เงื่อนไขที่เพิ่งสุกงอม (Stripe ไทยยัง payout บาทในประเทศเท่านั้น)

III
วงจรชาวสวน

เจ็ดขั้น ทุกปี

ชาวสวนผ่าน 7 ขั้นนี้ทุกปี. ไอเดียดั้งเดิมสองตัวเข้าถึงคนละขั้น — เลยต้องอยู่ในแอปเดียวกัน. แอปที่อยู่ทั้งปฏิทินคือเพื่อน ไม่ใช่เครื่องมือที่เปิดปีละครั้ง.

  1. ปลูก ที่ปรึกษาสายพันธุ์
    เลือกพันธุ์ผิด = เสีย 5 ปี (ต้นกาแฟ 3–4 ปีถึงเก็บเกี่ยว).
  2. ดูแล ที่ปรึกษา/ตรวจโรค
    ตัดแต่ง โรค ศัตรูพืช มองไม่เห็น.
  3. เฝ้าระวัง
    การระบาดของแมลง/โรคมองไม่เห็นจนสาย.
  4. เก็บเกี่ยว
    เก็บเร็ว/ช้าเกินไป กระทบ Brix และคุณภาพ.
  5. แปรรูป
    เลือกวิธี (washed/honey/natural) ผิด เสียโอกาสราคา.
  6. คัดเกรด ★ เครื่องสแกน (hero)
    ไม่รู้คะแนน → ตั้งราคาไม่ได้ เป็นผู้รับราคา.
  7. ขาย ★ marketplace (hero)
    ขายผ่านสหกรณ์/พ่อค้าเท่านั้น เข้าไม่ถึง roaster.

สร้างแยกกัน = แอปเปิดปีละครั้ง คนเลิกใช้เร็ว. รวมกัน = เพื่อนที่เปิดเป็นร้อยครั้งต่อปี.

IV
การโต้แย้ง

ทำไมการรวมจึงชนะ

3 เหตุผลที่ทบต้นกัน: รวมวงจร · วงจรสะสมข้อมูล · กำแพงหลายชั้น.

  • รวมวงจร — ใช้ที่ปรึกษาเดือนมีนา → มีแอปตอนเก็บเกี่ยวเดือนพฤศจิกา. ติดตั้งครั้งเดียว ใช้ 2 ผลิตภัณฑ์ ต้นทุนหาลูกค้าใหม่เป็นศูนย์.
  • วงจรสะสมข้อมูล — สแกน 1 ครั้ง = ข้อมูลติดป้าย 1 แถว. สายพันธุ์+GPS+คะแนน, ตำหนิ+วิธีแปรรูป+ราคา, ความชอบ roaster+โปรไฟล์ชาวสวน — แต่ละอย่างทำให้อีกอย่างแม่นขึ้น.
  • กำแพงหลายชั้น — แอปรวม = คู่แข่งต้องสร้างครบทุกโมดูลถึงจะแย่งได้. ต้นทุนแทนที่โตเร็วกว่าต้นทุนเพิ่มฟีเจอร์.

เครื่องสแกนเดี่ยวถูกลอกใน 6 เดือน — แต่ ชุดข้อมูล + ความสัมพันธ์ + วงจร ลอกไม่ได้.ข้อสังเกตเชิงกลยุทธ์

VII
ผลิตภัณฑ์

หก โมดูล

แต่ละโมดูลแก้ปัญหาขั้นเดียวของวงจร รวมกันแยกไม่ออก. 2 โมดูลเป็น hero — ตัว demo และตัวเปิดตลาด.

โมดูล 01 · ก่อนปลูก

ที่ปรึกษาพื้นที่ & สายพันธุ์

ปักหมุด GPS → ดึงความสูง/ความชัน/ดิน/อากาศ 20 ปี + พยากรณ์ IPCC AR6 (SSP2-4.5) → แนะนำสายพันธุ์พร้อมเหตุผล.

ตัวอย่าง: "แปลง 1,420 ม. ทิศ NE 22°C · แนะนำ Catimor (ทนสนิม) + Chiang Mai 80 (พันธุ์ไทย) เป็นหลัก, เก็บ Typica ไว้เป็น heritage/คุณภาพ · ต่ำกว่า ~1,300 ม. เสี่ยงเสื่อมภายในปี 2050."

โมดูล 02 · ฤดูปลูก

ตรวจโรค & ศัตรูพืช

ถ่ายรูปใบ → CV จำแนกใบสนิม/มอดเจาะผล/anthracnose + แผนที่ระบาดระดับลุ่มน้ำ. TFLite 4MB ทำงาน offline.

โมดูล 03 · เก็บเกี่ยว

ตัวจับเวลาเก็บเกี่ยว

ถ่ายเชอร์รี → วัดความสุก + พยากรณ์ 7 วัน "เก็บอังคาร ฝนพฤหัสจะลด Brix" + แนะวิธีแปรรูปตามตลาด.

★ hero · โมดูล 05 · ขาย

Marketplace ขายตรง roaster

ผลผลิตกลายเป็น "การ์ดล็อต" — รูป/วิดีโอเรื่องเล่า/รายงาน QC/ใบรับรอง. roaster ทั่วโลกค้นหา ขอตัวอย่าง ต่อรอง จ่ายผ่าน escrow. ระบบช่วยตั้งราคาเทียบตลาดโลก — ชาวสวนที่พูดอังกฤษไม่ได้ก็ต่อรองได้เหมือนมืออาชีพ.

โมดูล 06 · การตลาด

นักเล่าเรื่องชาวสวน

อัดเสียง 30 วิ ภาษาอาข่า/ม้ง/กะเหรี่ยง/คำเมือง → LLM แปลเป็นข้อความการตลาดอังกฤษ/จีน/ญี่ปุ่น + reel แนวตั้ง. เรื่องเล่า = premium ที่ผู้ซื้อ specialty ยอมจ่าย.

X
เงิน

โมเดลธุรกิจ

ค่าสมาชิกสองฝั่ง + ค่าธรรมเนียม marketplace + ทุนร่วม. ฉบับ v2 แก้หน่วยที่ผิดและตัดสมมติฐานที่เกินจริง.

  • B2C — ชาวสวน ฿99/เดือน Pro · ฟรี 5 สแกน/เดือน → Pro สแกนไม่จำกัด. หมายเหตุ: งานวิจัย WTP 2025 พบ median ชาวสวน <$2.4/เดือน (฿99 ≈ $2.75 เกิน median) → ตั้ง conversion จริง 10–15% ไม่ใช่ 30%.
  • B2B — roaster ฿990/เดือน + ค่าธรรมเนียม 8% · take-rate 8% แข่งขันได้ (เทียบ B2B โภคภัณฑ์จริง ~0.8–9.5%) — ไม่ใช่ "ลดจาก 12–15%" อย่างที่ v1 อ้าง.
  • B2G — ทุนร่วม · ทุน non-dilutive (NIA ≤5M฿, depa ≤1M฿, PMUC) = รายได้ก้อนแรกที่ realistic. carbon: $50–200 คือ ต้นทุน MRV ไม่ใช่รายได้ — รายได้จริง $5–45/ฟาร์ม/ปี ทำได้เฉพาะ aggregated.

Unit economics — ติดดิน

Roaster ARPU~$327/ปี (฿990×12)
v1 พิมพ์ "$990 LTV"ผิดหน่วย ~3×
LTV ชาวสวน (sub ล้วน)~$30–100
$350 LTVเฉพาะเมื่อรวม marketplace GMV
CAC (co-op/word-of-mouth)~$100

LTV/CAC จะดีต่อเมื่อ input ข้างบนเป็นจริง — ไม่ใช่ค่าตั้งต้น. recurring SaaS ARR ต้องแยกบรรทัดจาก marketplace GMV เลข "$1M ARR" รวมของ v1 ไม่ decompose จาก unit จริง.

XII
การป้องกัน

5 ชั้นกำแพง

แต่ละชั้นเดี่ยวไม่พอ การรวมกันทำให้การ clone ไม่คุ้มค่า.

  • ชุดข้อมูลเฉพาะไทย — โมเดล CV เป็นของสามัญแล้ว กำแพงจริงคือ data เฉพาะถิ่น (Catimor/Typica/Geisha ดินภูเขาไฟดอยช้าง) ที่ชุดต่างชาติทำได้ไม่ดี ทบต้นทุกปี.
  • ความไว้ใจชาวเขาผ่านช่องทาง — Hillkoff + มูลนิธิแม่ฟ้าหลวง (มรดก 30+ ปี) ผู้เล่นต่างชาติเข้าไม่ถึง.
  • เสียง+เล่าเรื่องหลายภาษา — อาข่า/ม้ง/กะเหรี่ยง/คำเมือง ไม่มี SaaS โลกไหน localize. ได้เปรียบ 3–5 ปี.
  • รวมวงจรผูกผู้ใช้ — สายพันธุ์ + ประวัติหลายฤดู + ความสัมพันธ์ roaster อยู่ในแอป → ต้นทุนเปลี่ยนสูง.
  • เครือข่ายสองด้าน — ชาวสวนเยอะ→ดึง roaster, roaster เยอะ→ราคาขึ้น ดึงชาวสวน. คุมฝั่งของก่อน.
